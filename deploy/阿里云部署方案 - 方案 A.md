## 一、架构设计

### 1.1 整体架构

```
┌─────────────────────────────────────────────────────────────────┐
│                         用户访问层                               │
│                    (Browser / Mobile App)                       │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│                      CDN 加速层 (可选)                           │
│                   阿里云 CDN + OSS 加速                           │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌───────────────┐    ┌──────────────────┐    ┌───────────────────┐
│   前端静态资源  │    │   后端 API 服务    │    │     数据库层       │
│  OSS 对象存储   │◄──►│ 轻量应用服务器   │◄──►│  RDS MySQL 基础版  │
│  (HTML/JS/CSS)│    │    (SWAS)        │    │   (2 核 2G 50GB)   │
└───────────────┘    └──────────────────┘    └───────────────────┘
                              │
                              ▼
                   ┌───────────────────┐
                   │   对象存储 OSS     │
                   │  (文件/图片上传)   │
                   └───────────────────┘
```

### 1.2 服务选型说明

| 层级 | 推荐服务 | 选型理由 | 备选方案 |
|------|----------|----------|----------|
| **前端** | OSS 静态托管 + CDN | 成本最低（按量付费），无需管理服务器，天然支持 HTTPS | 轻量应用服务器（需自行配置 Nginx） |
| **后端** | 轻量应用服务器 SWAS | 性价比高，开箱即用，含公网 IP，支持一键部署镜像 | 函数计算 FC（按需付费，但冷启动有延迟） |
| **数据库** | RDS MySQL 基础版 | 稳定可靠，自动备份，支持升级高可用版 | MongoDB 副本集（适合文档型数据） |
| **文件存储** | OSS 标准存储 | 与前端同源，便于统一管理，支持图片处理 | 轻量应用服务器本地盘（容量有限） |

### 1.3 网络架构

```
互联网 ──► 公网 IP(SWAS) ──► Docker 容器 (后端服务)
              │
              └──► 内网访问 RDS MySQL (同一地域，免流量费)
```

**网络优化建议**：
- SWAS 与 RDS 选择**同一地域**（如：华东 1-杭州），内网互通，流量免费
- OSS 选择**同地域**，内网上传加速，降低延迟
- 开启 RDS 白名单，仅允许 SWAS 内网 IP 访问

## 三、部署步骤

### 3.1 准备工作

#### 3.1.1 注册与实名认证
1. 访问 [阿里云官网](https://www.aliyun.com/) 注册账号
2. 完成**实名认证**（个人/企业均可）
3. 绑定支付宝或银行卡（用于充值）

#### 3.1.2 选择地域
**推荐**：华东 1（杭州）  
理由：离用户近、节点丰富、与大多数服务同地域

#### 3.1.3 创建资源组（可选）
便于后续资源管理与成本统计

---

### 3.2 步骤一：创建轻量应用服务器（SWAS）

#### 3.2.1 进入购买页面
访问 [轻量应用服务器](https://www.aliyun.com/product/swas)

#### 3.2.2 选择配置
```
地域：华东 1（杭州）
镜像：应用镜像 → Docker（推荐）或 系统镜像 → Ubuntu 22.04
套餐：2 核 2G 50GB SSD
带宽：3Mbps 起步（可后续升级）
购买时长：3 个月起（首购建议 1 年享折扣）
```

#### 3.2.3 设置登录凭证
- 创建**密钥对**（推荐）或设置密码
- 下载私钥文件 `.pem`，妥善保管

#### 3.2.4 完成购买
确认订单并支付，等待 1-3 分钟创建完成

#### 3.2.5 获取连接信息
在控制台获取：
- 公网 IP 地址
- SSH 端口（默认 22）
- 用户名（`root` 或镜像预设）

---

### 3.3 步骤二：配置后端服务

#### 3.3.1 SSH 登录服务器
```bash
# 使用密钥登录
ssh -i /path/to/your-key.pem root@<公网 IP>

# 或使用密码登录
ssh root@<公网 IP>
```

#### 3.3.2 安装 Docker（如镜像未预装）
```bash
# 一键安装 Docker
curl -fsSL https://get.docker.com | bash -s docker

# 设置开机自启
systemctl enable docker
systemctl start docker

# 验证安装
docker --version
```

#### 3.3.3 部署后端应用
```bash
# 创建应用目录
mkdir -p /opt/app
cd /opt/app

# 上传代码或拉取 Git 仓库
git clone <your-repo-url> .

# 或使用 Docker Compose（推荐）
# 创建 docker-compose.yml
cat > docker-compose.yml << EOF
version: '3.8'
services:
  api:
    image: your-backend-image:latest
    ports:
      - "8080:8080"
    environment:
      - DB_HOST=<RDS 内网地址>
      - DB_PORT=3306
      - DB_USER=root
      - DB_PASSWORD=<密码>
      - DB_NAME=your_db
    restart: always
    volumes:
      - ./logs:/app/logs
EOF

# 启动服务
docker-compose up -d

# 查看日志
docker-compose logs -f
```

#### 3.3.4 配置防火墙
在阿里云控制台 → 轻量应用服务器 → 防火墙：
```
放行端口：
- 80 (HTTP)
- 443 (HTTPS)
- 8080 (后端 API，按需)
```

---

### 3.4 步骤三：创建 RDS MySQL 数据库

#### 3.4.1 进入 RDS 购买页面
访问 [RDS MySQL](https://www.aliyun.com/product/rds/mysql)

#### 3.4.2 选择配置
```
地域：华东 1（杭州）【必须与 SWAS 同地域】
版本：MySQL 8.0 或 5.7
系列：基础版（单节点）
规格：2 核 2G
存储：50GB SSD（可自动扩容）
网络类型：专有网络 VPC
交换机：与 SWAS 同一 VPC（自动匹配）
```

#### 3.4.3 设置账号密码
- 创建高权限账号
- 设置强密码（建议 16 位以上，含大小写+数字+符号）

#### 3.4.4 配置白名单
在 RDS 控制台 → 白名单与安全组：
```
添加 SWAS 的内网 IP 地址（或 0.0.0.0/0 临时测试，生产环境请限制）
```

#### 3.4.5 获取连接信息
- 内网地址：`rm-xxxxx.mysql.rds.aliyuncs.com`
- 端口：`3306`
- 数据库名：自行创建
- 账号密码：步骤 3.4.3 设置

#### 3.4.6 初始化数据库
通过 SWAS 连接 RDS 执行初始化：
```bash
# 安装 MySQL 客户端
apt-get update
apt-get install -y mysql-client

# 连接数据库
mysql -h <RDS 内网地址> -u root -p

# 执行初始化 SQL
source /opt/app/init.sql
```

### 3.5 步骤四：配置 OSS 对象存储

#### 3.5.1 创建 Bucket
访问 [OSS 控制台](https://oss.console.aliyun.com/)：
```
Bucket 名称：全局唯一（如：your-app-2026）
地域：华东 1（杭州）
读写权限：私有（推荐）或 公共读
存储类型：标准存储
```

#### 3.5.2 上传前端静态文件
```bash
# 安装 OSS 命令行工具 ossutil
wget http://gosspublic.alicdn.com/ossutil/1.7.18/ossutil64
chmod 755 ossutil64
./ossutil64 config  # 配置 AccessKey

# 上传文件
./ossutil64 cp -r ./dist oss://your-app-2026/
```

#### 3.5.3 配置静态网站托管（可选）
在 OSS 控制台 → Bucket → 静态页面：
```
默认首页：index.html
默认 404 页：404.html
```

#### 3.5.4 获取访问域名
- 外网 endpoint：`your-app-2026.oss-cn-hangzhou.aliyuncs.com`
- 绑定自定义域名后可通过 CDN 加速

### 3.6 步骤五：配置域名与 HTTPS

#### 3.6.1 备案域名
- 国内服务器必须备案
- 通过阿里云代备案系统提交

#### 3.6.2 申请免费 SSL 证书
访问 [SSL 证书服务](https://www.aliyun.com/product/cas)：
```
证书类型：免费 DV 证书
域名数量：1 个
验证方式：DNS 验证（推荐）或 文件验证
```

#### 3.6.3 部署证书
```bash
# 下载证书到服务器
scp cert.pem root@<SWAS_IP>:/etc/ssl/
scp key.pem root@<SWAS_IP>:/etc/ssl/

# Nginx 配置示例
cat > /etc/nginx/conf.d/your-app.conf << EOF
server {
    listen 443 ssl;
    server_name your-domain.com;
    
    ssl_certificate /etc/ssl/cert.pem;
    ssl_certificate_key /etc/ssl/key.pem;
    
    location / {
        proxy_pass http://localhost:8080;
        proxy_set_header Host \$host;
        proxy_set_header X-Real-IP \$remote_addr;
    }
}

server {
    listen 80;
    server_name your-domain.com;
    return 301 https://\$server_name\$request_uri;
}
EOF

# 重启 Nginx
nginx -t
systemctl restart nginx
```

## 五、常见问题

### Q1: 轻量应用服务器和 ECS 有什么区别？
**答**：轻量应用服务器（SWAS）是简化版 ECS，预装常用环境，适合快速部署。ECS 更灵活但需手动配置。初期建议 SWAS，后期可迁移至 ECS。

### Q2: RDS 基础版和高可用版有什么区别？
**答**：基础版是单节点，无自动故障切换；高可用版是主备架构，自动故障切换，可用性 99.95%。初期验证用基础版，生产环境建议高可用版。

### Q3: OSS 如何实现 HTTPS 访问？
**答**：方案一：绑定自定义域名并配置 SSL 证书；方案二：通过 CDN 加速，在 CDN 层配置 HTTPS（推荐）。

### Q4: 如何备份数据？
**答**：
- RDS：自动备份（保留 7-730 天可配置）+ 手动备份
- 代码：Git 版本控制 + 定期推送远程仓库
- OSS：开启版本控制和跨区域复制

### Q5: 预算超支怎么办？
**答**：
1. 在阿里云控制台设置**月度预算告警**
2. 开启**资源用量监控**
3. 释放闲置资源
4. 使用**成本分析**工具定位高消费项

## 七、附录：一键部署脚本（参考）

```bash
#!/bin/bash
# deploy.sh - 一键部署脚本（示例）

set -e

# 配置变量
APP_NAME="your-app"
DB_HOST="rm-xxxxx.mysql.rds.aliyuncs.com"
DB_USER="root"
DB_PASSWORD="your-password"
DB_NAME="your_db"

echo "🚀 开始部署..."

# 1. 更新系统
apt-get update && apt-get upgrade -y

# 2. 安装 Docker
curl -fsSL https://get.docker.com | bash -s docker
systemctl enable docker && systemctl start docker

# 3. 创建应用目录
mkdir -p /opt/$APP_NAME
cd /opt/$APP_NAME

# 4. 拉取代码
git clone <your-repo-url> .

# 5. 创建 .env 文件
cat > .env << EOF
DB_HOST=$DB_HOST
DB_USER=$DB_USER
DB_PASSWORD=$DB_PASSWORD
DB_NAME=$DB_NAME
EOF

# 6. 启动服务
docker-compose up -d

# 7. 查看状态
docker-compose ps
docker-compose logs -f

echo "✅ 部署完成！"
```
