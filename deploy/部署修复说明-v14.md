# 🛠️ 部署失败修复说明 v1.4

## ❌ 上次失败原因

### 问题 1：Python 语法错误
```
npm error SyntaxError: invalid syntax
npm error if flags := os.environ.get(env_name) or []:
```

**原因**：
- `better-sqlite3` 编译需要 `node-gyp`
- `node-gyp` 使用 Python
- 服务器默认 Python 版本太旧（< 3.8），不支持海象运算符 `:=`

**解决**：
- ✅ 安装 Python 3.8+
- ✅ 配置 npm 使用正确的 Python 版本：`npm config set python python3.8`

---

### 问题 2：PM2 权限错误
```
npm error Error: EACCES: permission denied, mkdir '/usr/lib/node_modules/pm2'
```

**原因**：
- admin 用户没有权限写入系统目录 `/usr/lib/node_modules/`

**解决**：
- ✅ 使用 `--unsafe-perm` 参数
- ✅ 或使用用户级安装：`npm install -g pm2 --prefix ~/.npm-global`

## 🔧 如果仍然失败

### 查看日志

```bash
# 查看 PM2 日志
su - admin -c "pm2 logs gig-api --lines 100"

# 查看 npm 安装日志
cat /home/admin/.npm/_logs/*.log | tail -50
```

### 手动修复（如果自动脚本失败）

```bash
# 1. root 登录
ssh root@106.14.172.39

# 2. 安装 Python 3.8
yum install -y python38 python38-pip python38-devel

# 3. 切换到 admin
su - admin

# 4. 配置 npm
cd /home/admin/flex-platform/backend
npm config set registry https://registry.npmmirror.com
npm config set python python3.8

# 5. 清理并重装
rm -rf node_modules package-lock.json
npm install --build-from-source --production --unsafe-perm

# 6. 启动服务
pm2 start --name gig-api npm -- start
```
