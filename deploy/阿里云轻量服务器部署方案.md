# 阿里云轻量应用服务器部署方案（隔离部署版）

> **服务器信息**
> - 公网 IP: `106.14.172.39`
> - 操作系统：Alibaba Cloud Linux 3.2104
> - 配置：2 核 2G
> - 部署原则：**完全隔离，不影响现有服务**

---

## 一、隔离设计

### 1.1 用户隔离
创建独立部署用户，与现有服务隔离：
```bash
# 创建部署用户
useradd -m -s /bin/bash giguser
# 设置密码（仅用于系统管理）
```

### 1.2 目录隔离
所有文件部署在独立目录：
```
/home/giguser/flex-platform/
├── backend/          # 后端代码
├── frontend/         # 前端构建产物
├── nginx/            # Nginx 配置
├── pm2/              # PM2 配置
└── logs/             # 日志目录
```

### 1.3 端口隔离
使用独立端口，避免冲突：

| 服务 | 端口 | 说明 |
|------|------|------|
| 前端访问 | **8080** | Nginx 前端服务 |
| 后端 API | **8081** | Node.js API 服务 |
| SSH | 22 | 远程管理 |

> ⚠️ **说明**: 8080/8081 端口不太可能与现有服务冲突

### 1.4 数据库隔离
使用 SQLite 文件数据库，无需额外端口：
```
/home/giguser/flex-platform/backend/gig_platform.db
```

## 二、部署步骤

### Step 1: 服务器准备

```bash
# 1. 连接到服务器
ssh root@106.14.172.39
# 密码：Cmb@20210101

# 2. 创建部署用户
useradd -m -s /bin/bash giguser
passwd giguser
# 设置密码：gig@Deploy2026

# 3. 创建部署目录
mkdir -p /home/giguser/flex-platform/{backend,frontend,nginx,logs,pm2}
chown -R giguser:giguser /home/giguser/flex-platform

# 4. 安装必要工具
yum install -y git nodejs npm nginx
```

### Step 2: 安装 Node.js（如果版本过低）

```bash
# 检查 Node.js 版本
node -v

# 如果版本 < 16，使用 NVM 安装新版本
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
source ~/.bashrc
nvm install 18
nvm alias default 18
```

### Step 3: 部署后端

```bash
# 切换到部署用户
su - giguser

# 进入后端目录
cd /home/giguser/flex-platform/backend

# 拉取代码
git clone https://github.com/Maoqi-zxy/gig-platform.git .
git checkout v1.0.0-snapshot

# 安装依赖
npm install --production

# 配置环境变量
cat > .env << EOF
NODE_ENV=production
PORT=8081
DATABASE_PATH=/home/giguser/flex-platform/backend/gig_platform.db
JWT_SECRET=gig-platform-aliyun-2026-deploy-key
CORS_ORIGIN=http://106.14.172.39:8080
EOF

# 安装 PM2
npm install -g pm2

# 启动服务
pm2 start --name gig-api npm -- start

# 保存 PM2 配置
pm2 save

# 设置开机自启
pm2 startup
# 按提示执行生成的命令
```

### Step 4: 部署前端

```bash
# 在前端目录构建
cd /home/giguser/flex-platform

# 拉取前端代码
git clone https://github.com/Maoqi-zxy/flexible-work-platform.git frontend-temp
cd frontend-temp

# 安装依赖
npm install

# 修改 API 地址
cat > .env.production << EOF
VITE_API_URL=http://106.14.172.39:8081
EOF

# 构建
npm run build

# 复制构建产物
cp -r ../dist /home/giguser/flex-platform/frontend/dist
chown -R giguser:giguser /home/giguser/flex-platform/frontend

# 清理临时目录
cd ..
rm -rf frontend-temp
```

### Step 5: 配置 Nginx

```bash
# 创建 Nginx 配置
sudo vim /etc/nginx/conf.d/gig-platform.conf
```

```nginx
# 前端服务
server {
    listen 8080;
    server_name 106.14.172.39;
    
    root /home/giguser/flex-platform/frontend/dist;
    index index.html;
    
    location / {
        try_files $uri $uri/ /index.html;
    }
    
    location /api {
        proxy_pass http://127.0.0.1:8081;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
    
    access_log /home/giguser/flex-platform/logs/nginx-access.log;
    error_log /home/giguser/flex-platform/logs/nginx-error.log;
}
```

```bash
# 测试 Nginx 配置
sudo nginx -t

# 重启 Nginx
sudo systemctl restart nginx
```

### Step 6: 配置防火墙

```bash
# 在阿里云控制台 → 防火墙 → 添加规则
# 或使用命令行（如有 firewall-cmd）
firewall-cmd --permanent --add-port=8080/tcp
firewall-cmd --permanent --add-port=8081/tcp
firewall-cmd --reload
```

### Step 7: 验证部署

```bash
# 检查后端服务
curl http://106.14.172.39:8081/api/health

# 检查前端服务
curl http://106.14.172.39:8080

# 检查 PM2 状态
pm2 status

# 查看日志
pm2 logs gig-api
```
