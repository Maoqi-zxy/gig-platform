# 🚨 部署失败修复指南

## 问题原因

**npm 安装失败原因**：`better-sqlite3` 需要原生编译，服务器上缺少必要的编译工具（gcc、g++、python）。

## 解决方案

### 方案 A：使用修复版部署脚本（推荐，最快）

**修复内容**：
- ✅ 自动安装编译工具（gcc-c++、python3、make）
- ✅ 配置 npm 淘宝镜像（加速下载）
- ✅ 使用 `--build-from-source` 参数

**部署命令**：

```bash
# root 登录 SSH
ssh root@106.14.172.39

# 下载修复版脚本
curl -o /tmp/deploy-fixed.sh https://raw.githubusercontent.com/Maoqi-zxy/gig-platform/main/deploy/deploy-root-to-admin-fixed.sh
chmod +x /tmp/deploy-fixed.sh

# 执行修复版部署
bash /tmp/deploy-fixed.sh
```

---

### 方案 B：手动编译修复（如果方案 A 仍失败）

**步骤 1：root 登录安装编译工具**
```bash
ssh root@106.14.172.39

# 安装编译工具
yum install -y gcc-c++ python3 make git
```

**步骤 2：切换到 admin 用户重新编译**
```bash
su - admin
cd /home/admin/flex-platform/backend

# 清理 node_modules
rm -rf node_modules package-lock.json

# 配置 npm 镜像
npm config set registry https://registry.npmmirror.com

# 重新安装（带编译）
npm install --build-from-source --production
```

**步骤 3：如果编译仍失败，使用 MongoDB 版本**

由于 SQLite 编译问题，建议改用 MongoDB（无需编译）：

```bash
cd /home/admin/flex-platform/backend

# 修改 package.json
sed -i 's/"better-sqlite3":.*/"mongodb": "^6.3.0",/' package.json

rm -rf node_modules package-lock.json
npm install --production

# 配置 MongoDB 连接字符串（使用免费的 MongoDB Atlas）
cat > .env << EOF
NODE_ENV=production
PORT=8081
MONGODB_URI=mongodb+srv://<user>:<pass>@cluster.mongodb.net/flex-gig
JWT_SECRET=gig-platform-aliyun-2026
CORS_ORIGIN=http://106.14.172.39:8080
EOF
```

---

### 方案 C：使用预构建镜像（最终方案）

如果编译问题持续存在，使用 Docker 部署：

```bash
# root 安装 Docker
yum install -y docker
systemctl start docker
systemctl enable docker

# 使用预构建镜像
docker run -d --name gig-platform \
  -p 8080:80 \
  -v /home/admin/flex-platform/data:/app/data \
  -e JWT_SECRET=your-secret \
  registry.cn-hangzhou.aliyuncs.com/your-namespace/gig-platform:latest
```

## 快速诊断

**检查编译工具是否安装**：
```bash
gcc --version
g++ --version
python3 --version
```

**检查 npm 配置**：
```bash
npm config get registry
# 应该是：https://registry.npmmirror.com
```

**查看完整错误日志**：
```bash
su - admin
cd /home/admin/flex-platform/backend
npm install --build-from-source 2>&1 | tee install.log
# 查看 install.log 了解具体错误
```
